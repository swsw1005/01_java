인터페이스와 추상클래스
1. 인터페이스는 추상클래스의 일종이다

2. 인터페이스의 모든 메서드는 내용이 없는 추상 메서드만 선언할 수있다
 예: public void aa();//추상메서드

3. 추상클래스는 내용있는 메서드와 내용없는 추상메서드가 혼합되어있는 경우를 말한다

4. 인터페이스 접근제한자는 항상 public 이다(생략을 하더라도 public 이다)

5. 인터페이스 필드(변수)는 황상 static final이다( static final을 생략하더라도 static final이다)
  ex)   static final int MAX = 100;
        MAX = 10000; >>> ERROR. 값 변경 불가


6. 추상 플래스는 추상메서드가 하나 이상 존재 해야 한다

7. 추상 클래스는 객체생성을 못한다(왜냐면, 내용없는 추상 메서드를 호출할 수도 있기 때문에)

8. 생성자, static메서드, private 메서드는 추상화 될수 없다

9. 추상메서드는 상속 받아서 오버라이딩을 반드시 해야한다


10. 상속은 물려받는것.  인터페이스는 장착하는것
    개발자 extends IT직업군  implements 공인중개사, 한식조리사자격증 등등
